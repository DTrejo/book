How to Get Your Online Presence & Resume Ready to Impress People

  • Buy yourname.com
    • Use Github Help to learn how to set up a custom domain on Github Pages (or ask a friend). https://help.github.com/articles/using-a-custom-domain-with-github-pages/
    • I personally like dreamhost to register a domain b/c they don't try to upsell you like godaddy does. You should never pay more than ~13/year for a domain.
    • Use instantdomainsearch.com to find a name that closely matches your real name using some combination of firstname, lastname, your initials, your middle name, etc. Do not click on the links, because some registrars do “domain frontrunning” which is where they see you want a domain name, and buy it before you can. If you see a name for <$100 that is perfect, consider buying it, but talk to someone else who has bought domain names in the past before you do anything. DO NOT visit the domain to check what's there. Instead do a search on duckduckgo for “!whois yourname.com” to see who owns it. The price may go up if you visit it, because the owner will know more people want to buy it.
    • Set up your DNS with cloudflare. There is a free plan, do not pay. This way, you don't need to switch your DNS provider if you want to transfer a domain. Also the interface is nicer to use. Also it gives your github pages HTTPS (it's not totally secure, b/c cloudflare controls the certificate, but who cares).
  • Interlink
    • Yourname.com <> github
    • Yourname.com <> linkedin
    • Yourname.com <> twitter
    • Yourname.com <> etc
  • Write 1 paragraph (~4 sentences) about each of your projects and post it on your site along with a screenshot or youtube video if you have it. You can easily make videos using quicktime, I think. Link to the relevant github repo if possible. Link to the live project if possible. In the paragraph, consider answering these three questions:
    • What was the problem?
    • How did you solve it? Think: describe the app you made; technical complexity, maintainability, number of team-mates, business reasoning, product reasoning, design reasoning, technical reasoning, [ask me i can go on]
    • What were the results? Think: # users, money saved, time saved, money made, your responsibility, which contributions were yours (don't lie), how long it took you, why your co-workers liked it, [ask me and i can go on]
  • Tip: when making sites, if it is possible for it to be static, host it on github pages. That way, you never have to spend time updating it in the future when the free hosting service you used stops being free, or shuts down.
